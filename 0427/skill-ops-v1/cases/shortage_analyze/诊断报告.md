# shortage_analyze-v1 诊断报告

## 1. 本轮目标

本轮不是单纯再改一次 `shortage_analyze` 规则，而是使用增强后的 `skill-ops-v1` 来重新完成以下闭环：

1. trace 分析；
2. 错例诊断；
3. 规则/标注反事实比较；
4. 标注疑点排序；
5. 高频冲突模式挖掘；
6. 决定是否继续修改规则。

优化后的案例保存于 `./opt/shortage_analyze-v1`。

## 2. skill-ops-v1 相比上一版的算法增强

### 2.1 标注疑点排序

新增 `scripts/rank_label_issues.py`，对错例进行启发式打分，优先排序：

1. 相同输入但标注冲突的样本；
2. 标注包含目标标签、但 trace 明确显示规则未命中的样本；
3. 与主冲突模式重叠的样本。

### 2.2 高频冲突模式挖掘

新增 `scripts/mine_conflict_patterns.py`，支持通过派生表达式定义模式列，并统计：

1. 模式支持度；
2. 标注阳性数；
3. 预测阳性数；
4. 冲突缺口。

### 2.3 保留的基础算法

继续保留上一版已有能力：

1. 结构化 trace；
2. 标签集合比较；
3. 漏报/误报统计；
4. 重复样本冲突检测；
5. 规则/标注反事实分析。

## 3. 使用 skill-ops-v1 对案例的执行结果

对 `opt/shortage_analyze-v1/data.xlsx` 重跑后：

- 总样本数：167
- 精确一致样本数：126
- 精确一致率：75.45%
- 错例数：41

与上一版优化结果相比，规则层结果没有继续提升，但 v1 给出了更强的诊断证据，说明当前不应继续放宽规则。

## 4. v1 找到的关键证据

### 4.1 高频冲突模式非常集中

`mine_conflict_patterns.py` 对以下派生模式做统计：

1. `组合替代关系为空 = is_empty_list(col('组合替代关系'))`
2. `单一替代不足 = num('单一替代库存汇总') < num('总欠料数量')`

结果显示：

- `组合替代关系为空=True 且 单一替代不足=True`
- 支持样本数：116
- 标注包含 `替代交付异常` 的样本数：29
- 预测包含 `替代交付异常` 的样本数：0
- 冲突缺口：29

这说明这不是零散错例，而是一个非常集中的高频冲突簇。

### 4.2 标注疑点排序把同一类问题顶到了最前面

`rank_label_issues.py` 的结果里，Top 疑点几乎都属于同一类：

- 标注包含 `替代交付异常`
- 预测未命中 `替代交付异常`
- trace 明确显示 `替代交付异常` 规则未命中

其中最高分样本还叠加了：

- 相同输入存在不同标注

这进一步增强了“标注疑点”而不是“规则缺口”的判断。

### 4.3 反事实比较结果仍然非常明确

对于 `组合替代关系=[]` 且 `单一替代库存汇总 < 总欠料数量` 的样本：

#### 如果修改规则

- 一致样本数：126 -> 74
- 一致率：75.45% -> 44.31%
- 修复 28 条冲突样本
- 额外破坏 80 条原本一致的样本

#### 如果修改标注

- 一致样本数：126 -> 154
- 一致率：75.45% -> 92.22%
- 修复 28 条冲突样本
- 不破坏已有一致样本

## 5. 为什么 v1 没有继续修改规则

这是本轮最重要的结论。

`skill-ops-v1` 的新算法不是为了“逼出更多规则修改”，而是为了更可靠地判断：

1. 还有哪些地方值得改规则；
2. 哪些地方不该再改规则。

在这个案例中，v1 的新增算法反而更强地说明：

- `组合替代关系=[]` 但标注为 `替代交付异常` 的 29 条样本，应优先视为高置信度标注疑点；
- 不应继续把规则往这类样本上放宽。

所以，`opt/shortage_analyze-v1` 没有再做新的规则放宽，这是经过更强算法证据支持后的主动决策，而不是没有继续优化。

## 6. v1 的实际价值

虽然 `shortage_analyze-v1` 的规则结果没有再比上一版多提分，但 `skill-ops-v1` 相比上一版已经有了明显方法提升：

1. 从“能分析一个冲突”升级为“能自动排序一批高风险冲突样本”；
2. 从“人工注意到一个模式”升级为“能自动把高频模式挖出来”；
3. 从“凭阅读判断更像错标”升级为“用模式支持度 + 疑点排序 + 反事实收益一起证明更像错标”。

## 8. 结论

`skill-ops-v1` 已经把当前案例从“规则优化”推进到了“规则运营”阶段：

1. 能追踪；
2. 能分诊；
3. 能做反事实比较；
4. 能给标注疑点排序；
5. 能自动发现高频冲突模式。

在 `shortage_analyze-v1` 里，新增算法的核心价值不是继续改规则，而是更高置信度地证明：当前剩余的主冲突更像标注问题，而不是规则还应继续放宽。
