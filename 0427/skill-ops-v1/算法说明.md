# skill-ops-v1 算法说明

`skill-ops-v1` 在 `skill-ops` 的三层架构上，新增两层增强算法：

1. 标注疑点排序；
2. 高频冲突模式挖掘。

## 1. 继承的基础算法

保留 `skill-ops` 原有三层：

- Trace 层：逐规则执行追踪、字段归一化、约束校验；
- 诊断层：集合差分、分布统计、重复样本冲突检测、反事实对照分析；
- 优化层：规则补全、保守式规则优化。

## 2. 新增算法一：标注疑点排序

### 2.1 目标

不是所有错例都应该优先看。`rank_label_issues.py` 的目标是用启发式评分，把“最像错标”的样本排到前面。

### 2.2 当前评分信号

每条样本的疑点分数由以下信号组成：

1. 与预测结果存在标签冲突；
2. 与完全相同输入的其它样本标签冲突；
3. 标注包含目标标签，但 trace 中目标规则未命中；
4. 满足人工定义的高风险冲突模式。

### 2.3 算法性质

这本质上是一个基于规则的 `label issue scoring`，类似弱监督场景中的疑似错标排序，但当前版本还没有引入概率模型。

## 3. 新增算法二：高频冲突模式挖掘

### 3.1 目标

很多冲突不是单点，而是成簇出现。`mine_conflict_patterns.py` 用来把这些簇自动找出来。

### 3.2 输入

用户提供一组派生表达式，例如：

- `组合替代关系为空`
- `单一替代不足`
- `推荐基线高于原始基线`

脚本会先把这些表达式转成布尔派生列，再对这些列组合分组。

### 3.3 输出指标

每个模式会输出：

1. 支持样本数；
2. 标注包含目标标签的比例；
3. 预测包含目标标签的比例；
4. 冲突缺口。

### 3.4 算法性质

它本质上是一个轻量的 `pattern mining + group error analysis`。

## 4. 对 shortage_analyze 的意义

在 `shortage_analyze` 中，v1 的价值主要体现在：

1. 更快确认 `组合替代关系=[]` 是一个高频冲突模式；
2. 更快把这 29 条样本排序为高置信度标注疑点；
3. 让“改规则还是改标注”的判断有更强的样本簇级证据，而不是只靠单条样本阅读。

## 5. 仍可继续增强的方向

`skill-ops-v1` 仍有后续迭代空间：

1. 将启发式疑点分数升级为概率式错标评分；
2. 对候选规则做自动搜索，而不只是人工提供假设；
3. 引入多规则联合优化和阈值敏感性分析；
4. 为 trace 增加自动摘要和根因生成。
