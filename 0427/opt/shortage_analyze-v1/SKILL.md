# 欠料归因分析

对供应链欠料数据进行归因分析，识别8种L2异常类型。

## 快速使用

用户指定欠料原始Excel文件后，运行分析脚本：

```bash
python scripts/analyze_shortage.py <input_excel_path> [output_excel_path]
```

如需同时输出逐规则 trace，运行：

```bash
python scripts/generate_trace.py <input_excel_path> [output_excel_path] [trace_output_path]
```

分析完成后，输出文件中会新增 `L2分类结果` 列，包含匹配的L2标签（多个标签用 `、` 连接）。

## L2分类标签

按以下规则对每行数据进行匹配：

| 标签 | 判断条件 |
| --- | --- |
| 网容异常 | 网容 <= 0 或为空 |
| 用量异常 | M2 > M3-M13最大值 或 历史交易数量 > 原始基线 |
| 补库异常 | 历史交易数量 > 补库提前期补库和调拨汇总数量 |
| 基线异常 | 修改基线ROP < 推荐基线 或 历史交易数量 > 原始基线 或 (原始基线=0 且 历史交易数量=0 且 网容>0) 或 (历史需求为0且推荐基线高于原始基线) |
| 计划参数异常 | 最终补库提前期 < 计算补库提前期 |
| 补库供应不及时 | 补库在途时间 > 最终补库提前期 或 调拨在途时间 > 最终补库提前期 |
| 责任库房异常 | 责任库房预测物流时长 > SLA承诺时间 |
| 替代交付异常 | 单一替代库存汇总 < 总欠料数量 且 组合替代关系非空 且 (组合替代库存汇总为空/nan 或 组合替代库存逐项无法满足需求) |

详细规则说明见 [references/rules.md](references/rules.md)。

## Trace 输出

`generate_trace.py` 会输出 `jsonl` 文件。每行包含：

- `prediction`：当前脚本预测标签
- `answer`：样本中的人工标注
- `trace.<规则名>.conditions`：规则条件命中情况
- `trace.<规则名>.evidence`：关键字段值
- `trace.<规则名>.rationale`：规则解释

## 已知问题

验证结果显示训练数据中仍存在部分规则与标注冲突，尤其是 `组合替代关系=[]` 但标注为 `替代交付异常` 的样本。建议结合 `skill-ops` 的反事实分析脚本判断更可能是规则问题还是标注问题，再决定是否修改规则。
