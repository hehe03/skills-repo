# 欠料归因分析

对供应链欠料数据进行归因分析，识别8种L2异常类型。

## 快速使用

用户指定欠料原始Excel文件后，运行分析脚本：

```bash
python scripts/analyze_shortage.py <input_excel_path> [output_excel_path]
```

分析完成后，输出文件中会新增 `L2分类结果` 列，包含匹配的L2标签（多个标签用 `、` 连接）。

## L2分类标签

按以下规则对每行数据进行匹配：

| 标签 | 判断条件 |
|------|----------|
| 网容异常 | 网容 <= 0 或为空 |
| 用量异常 | M2 > M3-M13最大值 或 历史交易数量 > 原始基线 |
| 补库异常 | 历史交易数量 > 补库提前期补库和调拨汇总数量 |
| 基线异常 | 修改基线ROP < 推荐基线 或 历史交易数量 > 原始基线 或 (原始基线=0 且 历史交易数量=0 且 网容>0) |
| 计划参数异常 | 最终补库提前期 < 计算补库提前期 |
| 补库供应不及时 | 补库在途时间 > 最终补库提前期 或 调拨在途时间 > 最终补库提前期 |
| 责任库房异常 | 责任库房预测物流时长 > SLA承诺时间 |
| 替代交付异常 | 单一替代库存汇总 < 总欠料数量 且 组合替代关系非空 且 (组合替代库存汇总为空/nan 或 无法满足需求) |

详细规则说明见 [references/rules.md](references/rules.md)。

## 输入数据要求

Excel文件需包含以下关键字段（宽表格式）：

- 网容、M2、M3-M13最大值、历史交易数量、原始基线
- 补库提前期补库和调拨汇总数量、修改基线ROP、推荐基线
- 最终补库提前期、计算补库提前期
- 补库在途时间、调拨在途时间（格式：`62.87d`）
- 责任库房预测物流时长、SLA承诺时间（格式：`8.94h`）
- 单一替代库存汇总、总欠料数量、组合替代关系、组合替代库存汇总、组合替代需求数量

## 特殊字段格式处理

- **时间格式**：补库在途时间、调拨在途时间为"数字d"（天）；责任库房预测物流时长、SLA承诺时间为"数字h"（小时）
- **组合替代关系**：JSON数组格式，如 `[]` 或 `["item1", "item2"]`
- **空值处理**：数值比较时，任一值为空则该条件不满足

## 已知问题

验证结果显示准确率约73%，剩余问题主要由示例数据标注不一致导致：

1. **相同数据不同标签**：存在8组数据完全相同但answer列不同的情况
2. **替代交付异常不一致**：组合替代关系=[]且满足条件1的116行中，29行有标签，87行没有标签，无法区分
3. **基线异常不一致**：部分行条件相同但标签不同

建议在使用前检查数据标注的准确性。

## 分析流程

1. 读取用户指定的Excel文件（默认第一个sheet）
2. 对每一行数据，遍历所有L2规则进行匹配
3. 汇总所有匹配的标签，用 `、` 连接
4. 若无匹配标签，结果为 `- 未匹配到分支`
5. 在原数据基础上新增 `L2分类结果` 列，另存为新Excel文件
