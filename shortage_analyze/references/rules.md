# 欠料归因规则说明

## 概述

本文档详细描述欠料归因分析系统中使用的8条L2分类规则。每条规则对应一种可能的欠料原因，系统会根据数据特征自动匹配适用的规则。

---

## 规则1：网容异常

### 规则名称

网容异常

### 判断条件

- **条件1**：网容 <= 0 或为空

### 业务含义

网容（网络可用库存）为0或为空，表示当前无可用库存，可能导致欠料。

### 匹配逻辑

```
如果 网容 <= 0 或 网容为空 → 匹配网容异常
```

### 数据字段

- `网容`：网络可用库存数量

---

## 规则2：用量异常

### 规则名称

用量异常

### 判断条件

- **条件1**：M2（需求量） > M3-M13最大值（历史供应最大值）
- **条件2**：历史交易数量 > 原始基线

### 业务含义

当前需求量超过历史供应能力，或者历史用量超过基线设定，可能导致欠料。

### 匹配逻辑

```
如果 M2 > M3-M13最大值 → 匹配用量异常
或者 如果 历史交易数量 > 原始基线 → 匹配用量异常
```

### 数据字段

- `M2`：需求量/消耗量
- `M3-M13最大值`：历史13个月最大供应量
- `历史交易数量`：历史实际交易数量
- `原始基线`：原始设定的基线数量

---

## 规则3：补库异常

### 规则名称

补库异常

### 判断条件

- **条件1**：历史交易数量 > 补库提前期补库和调拨汇总数量

### 业务含义

历史交易用量超过补库计划量，补库不足以覆盖历史消耗。

### 匹配逻辑

```
如果 历史交易数量 > 补库提前期补库和调拨汇总数量 → 匹配补库异常
```

### 数据字段

- `历史交易数量`：历史实际交易数量
- `补库提前期补库和调拨汇总数量`：补库计划数量

---

## 规则4：基线异常

### 规则名称

基线异常

### 判断条件

- **条件1**：修改基线ROP < 推荐基线
- **条件2**：历史交易数量 > 原始基线
- **条件3**：原始基线 = 0 且 历史交易数量 = 0 且 网容 > 0

### 业务含义

基线设置不合理，包括：

- 基线设置过低（低于推荐值）
- 历史用量超过基线
- 有库存但基线设为0

### 匹配逻辑

```
如果 修改基线ROP < 推荐基线 → 匹配基线异常
或者 如果 历史交易数量 > 原始基线 → 匹配基线异常
或者 如果 原始基线=0 且 历史交易数量=0 且 网容>0 → 匹配基线异常（基线配置问题）
```

### 数据字段

- `修改基线ROP`：修改后的基线值
- `推荐基线`：系统推荐的基线值
- `历史交易数量`：历史实际交易数量
- `原始基线`：原始设定的基线数量
- `网容`：网络可用库存

---

## 规则5：计划参数异常

### 规则名称

计划参数异常

### 判断条件

- **条件1**：最终补库提前期 < 计算补库提前期

### 业务含义

最终设置的补库提前期小于系统计算值，可能导致补库计划不足。

### 匹配逻辑

```
如果 最终补库提前期 < 计算补库提前期 → 匹配计划参数异常
```

### 数据字段

- `最终补库提前期`：实际设置的补库提前期
- `计算补库提前期`：系统计算的补库提前期

### 时间格式说明

- 时间值格式：`62.87d`（天）或 `8.94h`（小时）
- 比较时自动将小时转换为天（除以24）

---

## 规则6：补库供应不及时

### 规则名称

补库供应不及时

### 判断条件

- **条件1**：补库在途时间 > 最终补库提前期
- **条件2**：调拨在途时间 > 最终补库提前期

### 业务含义

补库或调拨在途时间超过补库提前���，导致供应赶不上需求。

### 匹配逻辑

```
如果 补库在途时间 > 最终补库提前期 → 匹配补库供应不及时
或者 如果 调拨在途时间 > 最终补库提前期 → 匹配补库供应不及时
```

### 数据字段

- `补库在途时间`：补库运输时间
- `调拨在途时间`：调拨运输时间
- `最终补库提前期`：设置的补库提前期

### 时间格式说明

- 时间值格式：`62.87d`（天）或 `8.94h`（小时）
- 比较时自动将小时转换为天（除以24）

---

## 规则7：责任库房异常

### 规则名称

责任库房异常

### 判断条件

- **条件1**：责任库房预测物流时长 > SLA承诺时间

### 业务含义

责任库房的物流时长超过SLA承诺，可能导致迟到欠料。

### 匹配逻辑

```
如果 责任库房预测物流时长 > SLA承诺时间 → 匹配责任库房异常
```

### 数据字段

- `责任库房预测物流时长`：责任库房到目的地的物流时间
- `SLA承诺时间`：SLA承诺的物流时间

### 时间格式说明

- 时间值格式：`62.87d`（天）或 `8.94h`（小时）
- 比较时自动将小时转换为天（除以24）

---

## 规则8：替代交付异常

### 规则名称

替代交付异常

### 判断条件

- **条件1**：单一替代库存汇总 < 总欠料数量
- **条件2**：组合替代关系非空 且（组合替代库存为空/无法满足需求）

### 业务含义

替代方案库存不足，无法通过替代料满足欠料需求。

### 匹配逻辑

```
如果 单一替代库存 < 总欠料数量 且 有替代关系但替代库存不足 → 匹配替代交付异常
```

注意：

- 如果组合替代关系为空 `[]`，表示没有替代方案，不标记为替代交付异常

### 数据字段

- `单一替代库存汇总`：单一替代料的库存数量
- `总欠料数量`：欠料总数量
- `组合替代关系`：组合替代方案描述
- `组合替代库存汇总`：组合替代料的库存数量

---

## 规则优先级与组合

### 多规则匹配

一个欠料样本可能同时匹配多个规则，例如：

- 用量异常 + 基线异常
- 计划参数异常 + 补库供应不及时

### 结果格式

匹配结果用"、"分隔，例如：

- `用量异常、基线异常`
- `- 未匹配到分支`（无任何规则匹配）
