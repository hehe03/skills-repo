# 有监督方案：基于 train split 的特征原型分类

## 输入

脚本统一使用两个输入：

1. 包含 JSON 文件的文件夹。每个 JSON 文件是一个 trace 样本。
2. 元数据 CSV 文件，必须包含以下列：

```text
name,label,source,split
```

字段含义：

- `name`：JSON 文件名，例如 `case_001.json`。
- `label`：真实标签，支持 `goodcase` 和 `badcase`。
- `source`：trace 来源，用于分 source 统计。
- `split`：`train` 或 `test`。

有监督脚本固定使用 `split=train` 的样本训练，使用 `split=test` 的样本测试。训练集中必须至少包含一个 goodcase 和一个 badcase。

## 方法

脚本会把每个 trace 转成行为特征向量，包括任务数量、任务多样性、command 多样性、result 非空比例、result 长度、连续重复、循环模式、是否生成大纲、query 与任务名重合度等。

训练阶段：

- 对 train split 的特征做 robust normalization。
- 计算 goodcase 原型和 badcase 原型。

预测阶段：

- 对 test split 的每个样本计算到 goodcase 原型和 badcase 原型的距离。
- 使用距离差得到 goodcase 分数。
- 当 `score >= threshold` 时预测为 goodcase，否则预测为 badcase。

## 输出

脚本会在终端打印：

- 每个 trace 的实际标签和预测标签。
- 按 `source` 分组的 goodcase 正确数/总数、badcase 正确数/总数。
- badcase 作为正类时的混淆矩阵、precision、recall、F1-score。

同时会生成 markdown 指标文件，默认路径为：

```text
supervised_metrics.md
```

可通过 `--metrics-output` 指定。

## 使用方式

```powershell
python .\supervised\classify_supervised.py <json文件夹> <metadata.csv>
```

输出逐样本预测 TSV：

```powershell
python .\supervised\classify_supervised.py <json文件夹> <metadata.csv> --output predictions.tsv
```

调整分类阈值：

```powershell
python .\supervised\classify_supervised.py <json文件夹> <metadata.csv> --threshold 0.55
```

指定 markdown 指标文件：

```powershell
python .\supervised\classify_supervised.py <json文件夹> <metadata.csv> --metrics-output metrics.md
```
