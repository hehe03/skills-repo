# 无监督方案：行为特征评分 + bad-risk 优先拦截

## 输入

脚本统一使用两个输入：

1. 包含 JSON 文件的文件夹。每个 JSON 文件是一个 trace 样本。
2. 元数据 CSV 文件，必须包含以下列：

```text
name,label,source,split
```

字段含义：

- `name`：JSON 文件名，例如 `case_001.json`。
- `label`：真实标签，支持 `goodcase` 和 `badcase`。
- `source`：trace 来源，用于分 source 统计。
- `split`：`train` 或 `test`。

无监督脚本可以通过 `--split train` 或 `--split test` 指定评估集合；如果不指定，则使用 metadata 中的全部样本作为测试样本。

## 方法

脚本会对每个 trace 抽取行为特征，包括任务数量、任务名多样性、command 多样性、连续重复任务、连续循环模式、result 非空比例、result 长度、是否出现生成大纲、query 与任务名重合度等。

当前判断由两个通道组成：

- `behavior_prior`：样本像不像 goodcase。
- `bad_risk`：样本是否包含 badcase 风险信号。

最终分数：

```text
score = behavior_prior + centrality_weight * (centrality - 0.5) - bad_risk_weight * bad_risk
```

当 `bad_risk >= bad_risk_threshold` 时，优先判为 `badcase`。这样可以避免“有部分 good 信号，但执行行为明显异常”的样本被误判为 goodcase。

当一次输入不少于 3 个样本时，脚本会额外计算 `centrality` 群体一致性；样本太少时，`centrality` 显示为 `N/A`。

## 输出

脚本会在终端打印：

- 每个 trace 的实际标签和预测标签。
- 按 `source` 分组的 goodcase 正确数/总数、badcase 正确数/总数。
- badcase 作为正类时的混淆矩阵、precision、recall、F1-score。

同时会生成 markdown 指标文件，默认路径为：

```text
unsupervised_metrics.md
```

可通过 `--metrics-output` 指定。

## 使用方式

```powershell
python .\unsupervised\classify_unsupervised.py <json文件夹> <metadata.csv>
```

只评估 test split：

```powershell
python .\unsupervised\classify_unsupervised.py <json文件夹> <metadata.csv> --split test
```

输出逐样本预测 TSV：

```powershell
python .\unsupervised\classify_unsupervised.py <json文件夹> <metadata.csv> --output predictions.tsv
```

指定 markdown 指标文件：

```powershell
python .\unsupervised\classify_unsupervised.py <json文件夹> <metadata.csv> --metrics-output metrics.md
```

提升 badcase 召回时，可以先降低 bad-risk 阈值：

```powershell
python .\unsupervised\classify_unsupervised.py <json文件夹> <metadata.csv> --bad-risk-threshold 0.45
```
