# 无监督方案：行为特征评分 + 群体一致性校准

## 目标

在没有人工标注的情况下，读取 Agent trace 数据，输出每个样本的 `good case` 或 `bad case` 标签。

旧规则只检查“是否生成大纲”和“是否有明显报错”，容易误判。新的无监督方案不依赖单条硬规则，而是从 trace 的整体执行行为中抽取特征，再结合样本之间的一致性进行评分。

## 输入格式

脚本会自动检测输入路径：

- 如果输入是单个 `.json` 文件，则该文件是一个样本，样本名为文件名。
- 如果输入是单个 `.jsonl` 文件，则每一行是一个样本，样本名为行序号，例如 `1`、`2`、`3`。
- 如果输入是文件夹，则会读取其中的 `.json` 和 `.jsonl` 文件。
- 文件夹中的 `.json` 文件按“相对路径”作为样本名，例如 `a.json` 或 `sub/a.json`。
- 文件夹中的 `.jsonl` 文件按“文件名:行序号”作为样本名，例如 `data.jsonl:1`、`sub/data.jsonl:3`。

行序号从 1 开始。空行会被跳过。

## 核心思路

1. 抽取每条 trace 的行为特征。

   包括任务数量、任务名多样性、command 多样性、连续重复任务、连续循环模式、result 非空比例、result 长度、是否出现生成大纲、query 与任务名的词重合度等。

2. 计算行为质量先验分数。

   分数不是单条规则，而是多个弱信号的加权组合。例如 result 非空比例高、任务和 command 多样性高、最终结果较充分，是正向信号；连续重复、循环、任务缺少 command、产出过短，是负向信号。

3. 计算群体一致性分数。

   当一次输入至少 3 个可用样本时，脚本会对所有 trace 的特征做 robust normalization，并计算样本与主流行为的接近程度。这个分数用于缓和单条启发式的波动。

4. 合成最终分数。

   当样本数不少于 3 个时：

   ```text
   final_score = 0.75 * behavior_prior + 0.25 * centrality_score
   ```

   当只有 1 到 2 个样本时，群体一致性没有统计意义，脚本只使用：

   ```text
   final_score = behavior_prior
   ```

   当 `final_score >= threshold` 时输出 `good case`，否则输出 `bad case`。默认阈值是 `0.55`。

## 是否必须一次性输入所有测试样本

不绝对必须。脚本现在支持每次只输入一个样本，这时会只使用行为质量先验分数。

但更推荐批量输入一批同来源、同任务类型的样本。原因是无监督方案里的“群体一致性”需要样本之间互相比较，单样本无法判断它是否偏离主流行为。批量输入时，分数通常更稳定，也更容易发现异常执行路径。

实践建议：

- 快速在线判断单条 trace：可以单样本输入，结果作为初筛。
- 离线评估一批 trace：建议一次性输入整批样本。
- 如果未来要稳定地逐条判定，可以先用历史数据调好阈值，或保存一批历史正常样本作为参考分布。

## 优点

- 不需要标注数据，可以马上对历史 trace 做初筛。
- 不再依赖“报错文本”，适合 bad case 没有显式异常信息的情况。
- 输出包含 score 和 reason，方便人工复查边界样本。
- 只依赖 Python 标准库，Windows 环境可直接运行。

## 局限

- 无监督方案无法真正知道业务目标是否完成，只能基于行为形态估计质量。
- 如果数据集里大部分都是 bad case，群体一致性可能把常见坏模式当作正常模式，所以这里保留了行为质量先验作为主要分数。
- 阈值需要结合真实数据观察后微调。建议先查看分数落在 `0.45` 到 `0.65` 的样本，这些通常是最值得人工复核的边界 case。

## 使用方式

输入文件夹：

```powershell
python .\unsupervised\classify_unsupervised.py <trace文件夹>
```

输入单个 JSONL 文件：

```powershell
python .\unsupervised\classify_unsupervised.py .\data.jsonl
```

递归读取子目录：

```powershell
python .\unsupervised\classify_unsupervised.py <trace文件夹> --recursive
```

调整阈值：

```powershell
python .\unsupervised\classify_unsupervised.py <trace路径> --threshold 0.6
```

输出到 TSV 文件：

```powershell
python .\unsupervised\classify_unsupervised.py <trace路径> --output result.tsv
```

输出文件使用 `utf-8-sig` 编码，便于 Windows Excel 直接打开中文。
