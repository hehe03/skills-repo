# Agent trace 分析

## 背景

1、已有一个PPT生成Agent的运行trace数据；

2、该trace数据的字段为：

{query: XX

plan_list: [{task_name: XX, command: {name: XX, args: {arg1:XX, arg2:XX}}, task_id:XX, result:XX}, {task2 ...}]

}

## 目标

1、分析trace数据，快速将trace数据划分为good case和 bad case；

2、对于bad case，进一步分析原因，并提出改进建议；



## 规则

1、若trace中存在任务，其task_name为“生成大纲”，且result正常，则为goodcase，其它为badcase
